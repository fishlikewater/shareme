# 代码复用思考指引

## 何时使用

- 准备新增状态字符串、事件 kind、错误语义、DTO/snapshot、SQL、传输参数、文件路径规则。
- 准备复制 Wails 与 localhost agent 两边类似逻辑。
- 准备在前端组件内直接处理 API 入口差异。

## 判断

- 这个抽象是否已有稳定业务语义？
- Wails 桌面入口与 localhost agent 是否应该共享？
- 抽象后是否更容易测试，还是只隐藏差异？
- 是否已有包边界可承接：`app`、`protocol`、`localui`、`transfer`、`frontend/src/lib`？

## 最小动作

```bash
rg "相似关键字"
```

## 建议

- 入口差异优先收敛在适配层，业务状态复用同一类型。
- 两次重复不一定抽象；跨入口重复且语义一致时应优先复用。
- 先抽领域语义，不抽技术形状。
