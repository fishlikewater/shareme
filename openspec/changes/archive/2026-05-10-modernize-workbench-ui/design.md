## Context

当前前端是 React 18 + TypeScript + Vite 单页 UI，由 Wails 桌面入口和 localhost Web UI 兼容入口共同使用。展示组件已经按 `AppShell`、`DiscoveryPage`、`DeviceList`、`ChatPane`、`HealthBanner`、`TransferStatusBanner`、`PairCodeDialog` 拆分，API 适配差异被收敛在 `lib/api.ts`、`desktop-api.ts`、`localhost-api.ts`。

现有界面的问题主要在信息架构和视觉层级：大 hero 与统计卡片占用首屏，设备选择和会话发送不是第一视觉焦点；设备列表较宽但仍需用户阅读多个徽标判断状态；健康状态与传输状态以独立横幅堆叠，容易挤压主工作区。

本变更是前端可见行为与布局变更，属于 L1。它不改变后端运行时、Wails binding、localhost API、SQLite 或传输协议。

## Goals / Non-Goals

**Goals:**

- 将 UI 转为现代清爽的工作台式布局。
- 设备切换与设备状态判断成为首要路径。
- 通过可折叠设备 Dock 改善桌面窗口和窄屏空间利用。
- 会话、配对提示、不可达提示和发送入口在同一主工作区呈现。
- 保持 Wails 桌面和 localhost Web UI 共用同一前端组件与业务逻辑。

**Non-Goals:**

- 不新增或修改后端 API、事件字段、数据模型或本地存储。
- 不引入外部 UI 组件库。
- 不持久化用户布局偏好。
- 不重写消息排序、历史分页、传输状态计算或文件发送逻辑。

## Decisions

### Decision 1: 使用“设备 Dock + 主工作区 + 状态抽屉”

选择该结构替代当前“hero + 横幅 + 左右两栏”。

原因：
- 设备切换是传输工具最高频操作，Dock 形态能持续可见且占用空间少。
- 主工作区可优先容纳消息流和发送器，减少用户从展示区跳转到操作区的认知成本。
- 健康状态和传输状态移入状态抽屉后，正常情况下不会挤压会话空间，异常时仍可显著提示。

替代方案：
- 轻量美化现有两栏：成本低，但无法体现用户选择的现代清爽方向。
- 会话工作区最大化并把设备放右侧：沉浸感强，但会破坏当前从左到右选择设备的使用习惯。

### Decision 2: 折叠状态只保存在前端会话内

侧栏展开/收起由 `AppShell` 本地 state 管理，不写入配置文件。

原因：
- 当前需求是易用性和视觉优化，不需要新增配置模型。
- 避免扩大到桌面配置读写、localhost 兼容路径和迁移逻辑。
- 用户关闭后回到默认可解释状态，利于首次使用。

### Decision 3: 保持展示组件边界，优先重排组合与样式

`DeviceList` 继续负责设备列表，`ChatPane` 继续负责会话与发送，`HealthBanner` 与 `TransferStatusBanner` 继续负责状态展示，但视觉位置和样式改变。

原因：
- 现有测试已经围绕这些组件建立行为断言，保持边界可降低回归风险。
- API 差异仍停留在适配层，不渗入展示组件。
- 实现可分阶段完成：布局骨架、设备 Dock、主工作区、状态抽屉、响应式与视觉打磨。

### Decision 4: 视觉系统收敛为低装饰、高可读

使用浅灰白背景、青绿色状态色、橙色行动点、8-12px 圆角和轻阴影，取消大 hero 级标题和重渐变。

原因：
- 桌面传输工具需要长期停留和重复使用，低装饰能减少疲劳。
- 稳定尺寸和清晰焦点态能改善键盘与窄屏使用。
- 状态色用于传达在线、待配对、不可达和错误，但不作为唯一信息来源。

## Risks / Trade-offs

- 设备 Dock 收起后可解释性下降 -> 提供展开按钮、短名、状态文字和选中设备详情区。
- 完整工作台重构影响测试快照和角色查询 -> 优先保留按钮语义、aria 属性和关键文案，按行为更新测试。
- 状态抽屉可能隐藏传输信息 -> 活跃传输或错误状态自动提升摘要可见性，并保留可展开详情。
- 窄屏布局可能出现横向滚动 -> 使用固定轨道、`minmax(0, 1fr)`、稳定按钮尺寸和浏览器多宽度检查。
- 既服务 Wails 又服务 localhost -> 不使用宿主特有 UI 假设，所有布局逻辑留在 React 层。

## Migration Plan

1. 建立工作台布局骨架和折叠状态，不改 API。
2. 重构设备 Dock 展示与测试。
3. 重排会话主区、配对提示、不可达提示和发送器。
4. 重排健康/传输状态展示。
5. 完成响应式、焦点态、降低动效和视觉打磨。
6. 执行 `npm test`、`npm run build`，并进行浏览器宽度检查。

回滚策略：前端改动集中在 React 组件与 `styles.css`，若实现风险过大，可恢复到当前两栏布局并保留少量视觉 token 优化。

## Open Questions

- 是否需要在未来版本持久化侧栏折叠状态：本变更暂不做。
- 是否引入图标库提升 Dock 可识别性：本变更优先用文字、状态点和 CSS，若实现中现有表达不足再评估。
