# Message Share 现代工作台 UI 设计

## 背景

Message Share 的当前 React UI 采用大 hero、暖色渐变和多卡片布局，适合展示产品能力，但对高频桌面传输工具而言，核心任务路径不够直接。用户最常做的动作是判断设备状态、切换设备、配对、发送文字或文件、观察传输状态。

本设计将界面转为现代清爽的工作台：减少装饰，提升信息密度，让用户进入后直接操作。

## 目标

- 首屏直接呈现可操作工作区，而非展示型首页。
- 以设备状态判断和设备切换为第一优先级。
- 使用可折叠设备 Dock，兼顾新用户的可解释性和熟练用户的紧凑效率。
- 保持现有前后端契约不变，只调整前端布局、组件组合、样式和交互状态。
- 同时服务 Wails 桌面入口与 localhost Web UI。

## 非目标

- 不新增后端 API、Wails binding 或 localhost endpoint。
- 不改变消息、设备、配对、传输事件的数据结构。
- 不引入新 UI 组件库或图标库，除非实现阶段证明必要。
- 不做持久化布局偏好；侧栏折叠状态仅为当前前端会话状态。

## 信息架构

采用“设备 Dock + 工作台主区 + 状态抽屉”的三段结构：

- 左侧设备 Dock：默认可折叠。收起时显示短名、状态点和轻量提示；展开时显示完整设备名、配对状态、可达状态和最近消息。
- 中间工作台主区：顶部显示当前设备状态，主体是消息流，底部是发送器。未配对、不可达和空状态都在主区内提示。
- 右侧状态抽屉：承载健康状态与传输状态。默认克制显示，异常或传输活跃时提升可见性。
- 顶部应用栏：显示本机名、同步状态和设置入口，替代当前大 hero。

## 交互规则

- 点击设备项切换当前会话；设备项继续使用 `button` 和 `aria-pressed`。
- 侧栏折叠按钮需要明确 `aria-label`，折叠后仍能通过文字缩写和状态点识别设备。
- 未配对设备选中后，主区优先展示信任建立步骤和短码操作，发送器禁用。
- 不可达设备选中后，主区保留历史消息可见，同时显示不可达原因和重试提示。
- 有活跃传输时，状态抽屉显示摘要；用户可展开查看进度详情。
- 窄屏下设备 Dock 可横向或抽屉化呈现，切换设备后自动滚动到工作台主区。

## 视觉系统

- 色彩：浅灰白背景，青绿色表达在线/可用状态，橙色表达主行动，红色仅用于错误。
- 形状：主要容器使用 8-12px 圆角；避免当前 30px 大圆角和重阴影。
- 层级：主工作区最亮，设备 Dock 次亮，状态抽屉更克制；异常和活跃传输才抢视觉优先级。
- 字号：取消 hero 级标题；面板标题 18-22px，应用栏和状态信息 14-16px。
- 动效：只保留折叠、hover、状态更新的 150-220ms 微交互，并支持 `prefers-reduced-motion`。
- 响应式：确保 375、768、1024、1440 宽度无横向滚动，按钮文本不溢出。

## 验证

- `frontend/` 中执行 `npm test`。
- `frontend/` 中执行 `npm run build`。
- 浏览器手工检查 375、768、1024、1440 宽度。
- 重点路径：设备切换、未配对、不可达、发送文字、选择普通文件、选择并发送大文件、加载历史、传输状态展示。
