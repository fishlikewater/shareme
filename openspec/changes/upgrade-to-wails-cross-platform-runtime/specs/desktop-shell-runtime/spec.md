## ADDED Requirements

### Requirement: 系统必须以桌面窗口作为正式用户入口

系统 MUST 以 Wails 桌面窗口作为正式用户入口承载主界面。正式桌面构建启动后，用户 MUST 无需手动访问 `localhost` 地址、额外打开浏览器页面或单独管理本地 Web UI 进程。

#### Scenario: 启动桌面应用后直接进入主界面
- **WHEN** 用户启动正式桌面应用
- **THEN** 系统必须打开桌面主窗口并在窗口内加载 Message Share 主界面

#### Scenario: 正式入口不再依赖手动访问本地地址
- **WHEN** 正式桌面构建完成启动
- **THEN** 系统必须允许用户直接在桌面窗口内使用全部核心功能，而不得要求用户手动打开 loopback URL

### Requirement: 系统必须通过桌面桥接提供本地命令与事件

系统 MUST 通过桌面桥接向前端暴露本地命令与事件能力。该桥接 MUST 至少覆盖 Bootstrap、配对、文本发送、文件发送、极速文件发送、本地文件选择与历史消息分页查询，并且 MUST 向前端推送与当前应用状态模型兼容的事件载荷。

#### Scenario: 前端通过桌面桥接完成初始化
- **WHEN** 桌面主界面加载完成并请求初始化数据
- **THEN** 系统必须通过桌面桥接返回 Bootstrap 数据，而不是要求前端通过 loopback HTTP 拉取

#### Scenario: 前端通过桌面事件通道接收状态更新
- **WHEN** 配对、消息或传输状态发生变化
- **THEN** 系统必须通过桌面事件通道把状态更新推送到前端，并允许前端据此刷新界面

### Requirement: 系统必须协同管理桌面窗口生命周期与局域网运行时

系统 MUST 让桌面宿主统一管理窗口生命周期与局域网运行时生命周期。桌面应用启动时 MUST 初始化局域网发现、对等监听、存储和本地状态服务；桌面应用退出时 MUST 有序关闭这些资源，避免残留孤儿进程或损坏本地数据。

#### Scenario: 启动时同步拉起桌面界面与局域网运行时
- **WHEN** 用户启动桌面应用
- **THEN** 系统必须在显示主窗口的同时初始化局域网运行时，使设备发现、配对和发送能力可立即进入可用状态

#### Scenario: 退出时有序释放资源
- **WHEN** 用户关闭桌面应用
- **THEN** 系统必须有序关闭窗口、监听端口、数据库连接与后台服务，而不得留下独立运行的本地 UI 入口进程

### Requirement: 系统若提供 headless 兼容入口则必须复用同一运行时核心

系统若提供无窗口的 headless 兼容入口，该入口 MUST 复用与桌面正式入口相同的局域网运行时核心、配置目录布局与资源释放语义；该入口 MUST NOT 重新引入要求用户手动访问 `localhost` 的浏览器 UI 主入口，也 MUST NOT 冒充一个可独立完成首次配对与主动发送的本地交互入口。

#### Scenario: 以 headless 方式启动兼容入口
- **WHEN** 用户启动 `backend/cmd/message-share-agent` 构建出的 headless 兼容入口
- **THEN** 系统必须在不打开桌面窗口的前提下拉起局域网运行时，并使用与桌面正式入口一致的 `.message-share` 目录布局

#### Scenario: headless 兼容入口退出时释放同一套资源
- **WHEN** headless 兼容入口收到退出信号
- **THEN** 系统必须有序关闭局域网监听、发现、数据库与运行时资源，而不得留下 localhost 浏览器 UI 进程或额外守护进程
